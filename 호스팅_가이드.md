# 📱 모바일에서 카메라 스캔 가능하게 만들기

## 왜 카메라가 안 켜질까요?

모바일 크롬은 **보안 정책**상 다음 경우에만 카메라를 허용합니다:
- ✅ `https://...` 로 시작하는 페이지
- ✅ `localhost` (개발용)

따라서 다음은 카메라가 **차단**됩니다:
- ❌ 카카오톡 → 파일 클릭 (`content://...`)
- ❌ 파일관리자 → HTML 파일 더블클릭 (`file://...`)
- ❌ 첨부파일 다운로드 후 바로 열기

## ✅ 해결 방법

### 방법 1 — 블루투스 / USB OTG 바코드 스캐너 사용 (가장 추천)

**가장 안정적이고 빠른 방법입니다.** 카메라 없이도 즉시 사용 가능합니다.

1. 블루투스 바코드 스캐너 구매 (인터넷 10,000~30,000원대)
2. 핸드폰과 블루투스 페어링 (키보드처럼 인식됨)
3. 재고관리 프로그램의 **🔎 상품 검색** 칸을 한 번 탭(터치)
4. 스캐너로 상품 바코드를 찍으면 → 검색창에 자동 입력 + 엔터 → 상품 자동 선택
5. 입출고/위치/수량만 누르면 됩니다

> 💡 창고/매장 실무에서는 블루투스 스캐너가 카메라보다 훨씬 빠릅니다.

### 방법 2 — GitHub Pages에 무료 HTTPS 호스팅

한 번만 설정하면 영구적으로 모바일 카메라 스캔이 가능합니다.

**1단계: GitHub 가입 (1분)**
1. https://github.com 접속 → **Sign up**
2. 이메일, 비밀번호, 사용자명 입력

**2단계: 저장소 만들기 (1분)**
1. 우측 상단 **+** → **New repository**
2. Repository name: `inventory` (자유롭게)
3. **Public** 선택 (Private은 GitHub Pages 사용 불가)
4. **Add a README file** 체크
5. **Create repository** 클릭

**3단계: 재고관리.html 업로드 (1분)**
1. 만든 저장소에서 **Add file → Upload files**
2. `재고관리.html` 파일을 드래그해서 올리기
3. 파일명을 **`index.html`** 으로 변경 (중요!)
   - 업로드 후 파일을 클릭 → 연필 아이콘(편집) 옆 점3개(...) → Rename → `index.html`
4. **Commit changes** 클릭

**4단계: GitHub Pages 활성화 (30초)**
1. 저장소 상단 메뉴 **Settings**
2. 왼쪽 메뉴 **Pages**
3. Source: **Deploy from a branch**
4. Branch: **main** / **/(root)** → **Save**
5. 2~3분 대기

**5단계: URL 확인 및 사용**
1. Pages 페이지에 `https://(사용자명).github.io/inventory/` URL이 표시됨
2. 핸드폰 크롬에서 이 URL 접속
3. **카메라 권한 허용** (첫 접속 시 1회만)
4. ✅ 카메라 스캔 작동!
5. 핸드폰 홈 화면에 추가: 크롬 메뉴 → **홈 화면에 추가** → 앱처럼 사용

> ⚠️ 데이터는 브라우저에 저장되므로, GitHub Pages URL을 항상 같은 핸드폰의 같은 크롬으로 접속해야 데이터가 유지됩니다.
> 다른 기기에서 공유하려면 구글시트 연동을 꼭 설정하세요.

### 방법 3 — PC에서 사용 (Windows/Mac)

PC 크롬에서는 `file://` 로 열어도 일부 환경에서 작동합니다:
1. PC에 USB 바코드 스캐너 연결
2. `재고관리.html` 더블클릭으로 크롬에서 열기
3. **🔎 상품 검색** 칸 클릭 → 스캐너로 바코드 찍기
4. 카메라가 필요하면 PC 웹캠도 사용 가능

## 🔄 업데이트하는 방법 (GitHub Pages 사용 시)

프로그램이 업데이트되면:
1. GitHub 저장소 접속
2. `index.html` 파일 클릭
3. 연필 아이콘(편집)
4. 기존 내용 삭제하고 새 `재고관리.html` 내용 붙여넣기
5. **Commit changes**
6. 1~2분 후 모바일에서 새로고침

## ❓ 자주 묻는 질문

**Q. GitHub Pages가 너무 어려워요.**
- 블루투스 바코드 스캐너 + 직접입력으로도 충분히 빠르게 사용 가능합니다.
- 카메라 스캔이 꼭 필요하지 않다면 굳이 호스팅할 필요는 없어요.

**Q. PC에서는 카메라가 켜져요.**
- PC 크롬은 `file://` 보안 정책이 약간 다릅니다. 모바일은 더 엄격합니다.

**Q. 데이터를 어떻게 백업하나요?**
- 구글시트 연동을 꼭 설정하세요 (사용설명서 4번 항목)
- ⚙️ 설정 → 전체 백업으로 JSON 파일도 받아둘 수 있습니다
